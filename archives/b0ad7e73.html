2.5 绑定域名

如果你觉得上面的网站名字已经足够好了就省略下面的步骤吧, 如果你已经购买了域名,可以按照下面的步骤将github上的网站与之绑定.

2.5.1 解析域名:

首先要去 阿里云(或者你购买域名的其他云平台) 解析域名：

主机记录 记录类型 记录值
@ A 192.30.252.153
@ A 192.30.252.154
www CNAME 你的用户名.github.io
@ CNAME 你的用户名.github.io

主机记录就是域名前缀，常见用法有：

  • www：解析后的域名为 www.你的域名
  • @：直接解析主域名 你的域名
  • *：泛解析，匹配其他所有域名 *.你的域名

要指向空间商提供的 IP 地址，选择 类型 A ，要指向一个域名，选择 类型 CNAME

上面 192.30.252.153 , 192.30.252.154 就是GitHub提供的IP地址。

2.5.2 添加 CNAME

域名这边已经配置好，但还没结束，我们再来到HEXO博客的目录，进入source目录，新建一个CNAME文件，没有任何后缀名，编辑内容为：qiaofeng.xyz (这里替换为你购买的域名)

如果你之前已经部署到了 GitHub Pages 并开启了 HTTPS，那么直接在设置页面绑定你自己的域名，SSL/TLS 安全证书就会显示申请错误。

申请错误原因是：在验证域名所有权时会定位到 Github Pages 的主机上导致 SSL 证书申请失败。

正确的做法是：先去域名 DNS 把 GitHub 的解析暂停掉，然后再重新申请 SSL 证书，大约十秒左右就能申请成功，然后开启强制 HTTPS 访问。
